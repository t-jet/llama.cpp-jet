## What was wrong in the previous attempt

### Coverage (T114 / T115)

The primary failure was the wrong hybrid-mode denominator. The previous
attempt included `server-context.cpp` (3138 valid lines) in the denominator.
That file is the general server dispatcher; focused tests do not exercise its
HTTP handling code. The inclusion reduced the combined line rate from what
would have been a passing result to 21% (9785 / 45999):

```text
server-context.cpp: 3138 lines counted 8 times = 25104 out of 45999 total
almost all server-context.cpp lines were uncovered by focused tests
```

The second failure was separate per-test coverage without merging into a union.
OpenCppCoverage ran each test individually and produced 8 independent Cobertura
XMLs; the aggregate script summed the counts across separate runs, which
double-counts shared code. The correct approach uses binary `.cov` export per
test plus `--input_coverage` merge to produce one union result.

The third failure was omitting `llama-server` from the coverage measurement.
`server-cache-hybrid.cpp` has 1846 lines. Focused tests alone cover ~36% of
that file. The server integration paths (HTTP-driven cache lookup, store,
and restore) in that file are only exercised when the server runs with HTTP
requests. The HTTP probe was run but not tracked by OpenCppCoverage.

The LLVM build attempts (`cov-build.bat`, `cov-debug-build.bat`,
`cov-pdb-build.bat`, `cov-msbuild-build.bat`) were unnecessary. The build log
shows `lld-link: error: undefined symbol: __profc_...` because `lld-link`
ignores `-fprofile-instr-generate` as a linker flag on MSVC + clang-cl.
OpenCppCoverage (already found to work) was the correct fallback from the start.

## What was implemented this session

### `tools/server/tests/bench-cache-correctness.js`

Standard k6 script (no SSE extension required). Two scenarios:

- `warmup` (1 VU, 1 iteration): cold `/completion` request to seed the cache
  entry, measures `timings.cache_n` and `timings.prompt_ms`.
- `hit_probe` (1 VU, `BENCH_ITERS - 1` iterations): repeated identical
  request with `cache_prompt: true`, validates `timings.cache_n > 0`.

Custom metrics:

- `cache_miss_prompt_ms` (Trend): prompt processing time for cold requests.
- `cache_hit_prompt_ms` (Trend): prompt processing time for hit requests.
- `cache_n_tokens` (Trend): cache_n value per response.
- `cache_hit_rate` (Rate): fraction of probe iterations with `cache_n > 0`.

Correctness gate threshold: `cache_hit_rate >= 0.60`. k6 exits 1 if the gate
fails (T117 FAIL). The 60% floor allows for one or two early misses without
masking a genuine cache regression.

Environment variables: `BENCH_URL`, `BENCH_ITERS`, `BENCH_N_PREDICT`.

### `._design_docs/cache-handling-test-scripts/run_coverage.ps1`

Three-phase OpenCppCoverage runner:

1. Runs each of the 8 focused tests with `--export_type binary:<test>.cov`.
2. Starts `llama-server` under OpenCppCoverage with `--export_type binary:server-http-probe.cov`, waits for `/health`, sends 6 completion requests (1 cold + 5 warm), stops the server. OpenCppCoverage writes the binary `.cov` on process exit via its debug loop.
3. Merges all `.cov` files with `--input_coverage` flags (dummy target `cmd /c exit 0`) and exports a single union Cobertura XML.

Hybrid-mode denominator (excluded: `server-context.cpp`, `server-context.h`):

| File | Role |
| --- | --- |
| `server-cache-hybrid.cpp/h` | main hybrid cache implementation |
| `server-cache-store-cold.cpp/h` | cold store |
| `server-cache-controller.cpp/h` | controller |
| `server-cache-graph.cpp/h` | branch graph |
| `server-cache-io-worker.cpp` | async IO worker |
| `server-cache-policy-lru.cpp` | LRU policy |
| `server-cache-legacy.h` | legacy header |
| 8 focused test `.cpp` files | test code (nearly 100% covered) |

Reports per-file line rates, combined rate, and PASS/FAIL against 80%.

### `._design_docs/cache-handling-test-scripts/run_benchmark_k6.ps1`

Orchestrates `bench-cache-correctness.js`:

1. Starts `llama-server` in hybrid mode.
2. Sends one warmup request to pre-seed the cache entry.
3. Snapshots `/metrics` before.
4. Runs k6 with the correctness script.
5. Snapshots `/metrics` after.
6. Stops the server.
7. Computes public Prometheus counter deltas (`cache_hits_total`, `cache_misses_total`).
8. Writes `evidence-summary.md` with T116 evidence classification.
