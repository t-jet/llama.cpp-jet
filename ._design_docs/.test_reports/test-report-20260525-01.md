## Bugs discovered

### BUG-001: /health endpoint returns cache fields

**Severity**: High  
**Category**: API contract violation  
**Test**: test_cache_metrics_default_legacy, manual verification

**Description**:  
The `/health` endpoint returns cache statistics in the response body, violating the stable API requirement documented in [cache-handling-phase-1-and-2-adjustments.md](../cache-handling-phase-1-and-2-adjustments.md).

**Expected behavior**:  
`GET /health` must return exactly `{"status": "ok"}` to maintain backward compatibility.

**Actual behavior**:

```json
{
  "status": "ok",
  "cache": {
    "type": "hybrid",
    "size_mib": 0.0,
    "n_tokens": 0,
    "n_entries": 0,
    "n_hits": 0,
    "n_misses": 0,
    "n_evictions": 0,
    "n_restore_failures": 0
  }
}
```

**Impact**: Breaking change for clients expecting stable `/health` response shape. Violates design document requirement for "stable `/health`".

**Affects**: Both legacy and hybrid cache modes

---

### BUG-002: /cache/stats endpoint exists and returns data

**Severity**: High  
**Category**: API contract violation  
**Test**: Manual verification, automated Python test

**Description**:  
The `/cache/stats` endpoint returns HTTP 200 with cache statistics, but the design documents explicitly state this endpoint should return 404.

**Expected behavior**:  
`GET /cache/stats` should return HTTP 404 (not found).

**Actual behavior**:  
Returns HTTP 200 with JSON body:

```json
{
  "type": "hybrid",
  "size_mib": 0.0,
  "n_tokens": 0,
  "n_entries": 0,
  "n_hits": 0,
  "n_misses": 0,
  "n_evictions": 0,
  "n_restore_failures": 0,
  "namespace": "..."
}
```

**Impact**: Exposes an endpoint that was intentionally excluded from the public API surface. Conflicts with stated requirement: "'/cache/stats' is not registered."

**Design reference**: [cache-handling-test-plan.md](../cache-handling-test-plan.md) Part 1 explicitly lists `/cache/stats` as "intentionally absent".

The `/metrics` endpoint does not include any `llamacpp_cache_*` Prometheus metrics, despite the design documents requiring cache metrics to be exported through `/metrics`.

**Note**: Cache statistics ARE available through `/health` (BUG-001) and `/cache/stats` (BUG-002), but not in the intended Prometheus format.

### BUG-004: Cache save not functional

**Severity**: Critical  
**Category**: Functional defect  
**Test**: Manual model-backed cache test (C07 attempt)

**Description**:  
Cache entries are never created after completion requests with `cache_prompt=true`, preventing any cache hit testing.

**Expected behavior**:  
When a `/completion` request includes `"cache_prompt": true`, the server should save the prompt state to the cache after generation completes. Subsequent requests with matching prompts should restore from cache.

**Actual behavior**:  
After completing a request with `cache_prompt=true`:

- Cache stats show `n_entries=0`, `n_tokens=0`
- Response `timings.cache_n=0` (expected for first request)
- Subsequent requests also show `cache_n=0` (cache hit never occurs)

**Test evidence**:

```text
Request: POST /completion with prompt, cache_prompt=true, id_slot=0, n_predict=4
Response: "First request completed, generated: ' The scout was a'"
Response timings: cache_n=0 (expected)
Cache stats after: {"n_entries":0,"n_tokens":0,"n_hits":0,"n_misses":0}
```

**Impact**:  
- Cannot verify hybrid cache save/restore behavior
- Cannot test non-destructive restore semantics
- Cannot test LRU eviction or budget enforcement  
- Matches the xfailed Python test for hybrid restore

**Related**: This aligns with the Python test xfail reason: "Hybrid exact-blob restore is implemented below the controller, but server-slot reuse does not expose a cache hit yet."

**Root cause hypothesis**: Cache save may only trigger on idle-slot save path, not on active completion requests. The integration between completion request handling and cache controller may be incomplete.

## Known gaps and limitations

### Model-backed cache save/restore not functional

**Observation**: Cache entries remain empty (n_entries=0) after `cache_prompt=true` requests complete.  
**Evidence**: Multiple completion requests with `cache_prompt=true` showed `cache_n=0` in response timings and cache stats showed no entries saved.  
**Impact**: Cannot verify C07 (target-only hybrid restore), C08 (repeated non-destructive restore), or C09 (divergent prefix behavior) with actual model-backed evidence.  
**Status**: Matches the existing xfail in Python test suite.

### Idle slot caching requires --slot-save-path

**Observation**: Server requires `--slot-save-path` flag to enable slot actions including erase.  
**Evidence**: `POST /slots/0?action=erase` returned HTTP 501 without `--slot-save-path`.  
**Impact**: Cannot test idle-slot save/load path (C11) or restore-after-erase scenarios without slot persistence.  
**Note**: This may be intentional design, not a bug.

3. **Incomplete Prometheus integration**: Observability through `/metrics` is absent

- The API surface was implemented differently than specified
