Setup gap (root cause, not a coverage-threshold failure):

1. Per-test `.cov` files written by the script's `Start-Process` form
   are 34-54 bytes each (header-only, module-name string with no
   coverage data). The merged Cobertura XML reports `lines-covered=0,
   lines-valid=0` and a 0 / 0 combined line rate.
2. OpenCppCoverage 0.9.9.0 (released 2021) prints
   `[warning] No modules were selected. Please check the values of
   --modules and --excluded_modules` at exit. Manually verified
   alternate `--modules` forms (explicit test exe path, explicit
   `llama-server-impl.dll` path, `bin\Release\*.dll` wildcard, full
   `bin\Release` directory, full list of all 7 DLLs) all reproduce
   the same 37-byte stub `.cov` and the same "No modules were
   selected" warning. See
   `coverage-manual-20260607-02.log` and per-test
   `*-manual.log.stderr` files for the manual probe transcripts.
3. The build configuration is coverage-eligible per part-15 Section
   2: `CMAKE_CXX_FLAGS_RELEASE=/Zi /Ob1 /O2 /EHsc`,
   `CMAKE_EXE_LINKER_FLAGS_RELEASE=/DEBUG:FULL`, no `/DNDEBUG` for
   Release, PDB files present alongside every DLL and test exe. The
   20260604-06 baseline run (T114=0.8553, T114a=0.7035) used the
   same flag set in a `build-cov` RelWithDebInfo configuration. The
   remaining difference between the 20260604-06 working build and
   the current build is `BUILD_SHARED_LIBS=ON` (current) vs the prior
   static link: with shared libs, the cache source lives in
   `llama-server-impl.dll` rather than being linked into the test
   exe, and OpenCppCoverage 0.9.9.0 does not resolve the DLL's
   module-name in the merged Cobertura XML.

This is a coverage infrastructure gap, not a coverage threshold
failure, so the rows are classified BLOCKED, not FAIL, per the
`distinguish Release-build coverage gap from Start-Process bug` rule
in qa.md.

Developer handoff to close the gap: either (a) configure a
`build-cov` RelWithDebInfo configuration with
`BUILD_SHARED_LIBS=OFF` and rerun the script (the path the
20260604-06 baseline used), or (b) update `run_coverage.ps1` to
add `llama-server-impl.dll` to the `--modules` list explicitly and
to a `module_dll_pattern` in the script's argument builder, or (c)
upgrade OpenCppCoverage to a release that supports the VS17 2022
shared-library PDB layout.
