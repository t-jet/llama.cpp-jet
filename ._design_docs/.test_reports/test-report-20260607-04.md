## 12. Self-improvement post-task review

Items for qa.md memory to consider:

- PowerShell `& $ScriptPath @ArgList` splat binds positionally on PS 5.1 when named-flag strings are passed as elements; positional [int] Port then receives a path string and throws ParameterBindingArgumentTransformationException. Workaround: use direct `& $ScriptPath -BuildDir X -OutDir Y` invocation.
- Get-ChildItem with prefix-only path in PS 5.1 dot-source context may produce unexpected path resolution; use `-Path <dir> -Filter <pattern>` to be safe.
- Write-Host output is not captured by `2>&1 | Out-String`; per-scenario live.log at top level is empty even on successful run. Evidence-summary.md and per-variant subdir artifacts are the real evidence.
- Server flags: llama-server requires --model to load model; without it, server enters router mode ("no model will be loaded in this process") and /metrics, /completion return 400 with "model name is missing from the request".
- Start-Process -Wait with -RedirectStandardOutput on a child that itself Start-Process's a long-running server can hang past the child's exit because the server's stdout handle is inherited and held until parent reads/closes. Direct foreground invocation in a single script with explicit cleanup is more reliable.
